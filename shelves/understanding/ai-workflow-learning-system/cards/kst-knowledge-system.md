# Concept Card: K/S/T Knowledge System

## One-line Insight

不要把事实、规则和流程混进一个巨型 prompt。把它们拆成 K/S/T。

## Meaning

K/S/T 是我整理 AI 工作知识的三层结构：

- K: Knowledge，事实与领域知识
- S: Standards，角色、边界、交付规范
- T: Tactics，流程、卡点、复盘和回流

## My Rule

每次沉淀经验时，先问它应该进入哪一层。

## Examples

| Layer | Good Asset |
| --- | --- |
| K | 概念解释、材料摘要、领域事实 |
| S | 禁止事项、验收标准、角色边界 |
| T | 工作流、检查清单、任务模板、handoff |

## Reuse

适用于个人知识库、团队协作规范、Codex 项目规则、Skill 设计。
