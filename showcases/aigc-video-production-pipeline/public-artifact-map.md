# Public Artifact Map

This map explains artifact categories without publishing the private source files.

This map lists the kinds of materials around the video. The source materials themselves stay outside the public package.

## Artifact Categories

| Category | Description |
| --- | --- |
| Creative brief | A structured description of the story goal, audience, tone, and constraints. |
| Shot plan | A table that breaks the story into reviewable production units. |
| Prompt archive | A controlled record of prompt intent, asset role, and revision context. |
| Visual assets | Candidate and selected images used to support video production. |
| Review documents | Human-readable materials used to explain the project and check readiness. |
| Review workbooks | Spreadsheet packages used to inspect production units, assets, prompts, and missing fields. |
| QA previews | Rendered previews used to catch visual or formatting problems before delivery. |

## Sample Policy

Public examples should use:

- fake shot names
- fake prompt fragments
- generic asset roles
- reconstructed diagrams
- summarized review logic

Public examples should avoid:

- original prompt tables
- private planning tables
- original submission forms
- full visual asset sets
- personal or team information
- unreleased video files
- file paths from the working machine
- private automation details

## Safe Example

```text
Shot Unit: Scene A
Purpose: establish location and emotional tone
Asset Role: background reference
Review Check: composition, continuity, visible defects
Status: selected for package review
```

This kind of example shows the production method while keeping the private creative material out of the repository.
